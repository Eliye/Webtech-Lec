LESSON 2: Hypertext Transfer Protocol (HTTP)

HTTP

  • application layer communications protocol used to access resources (hypertext/hypermedia) on the World Wide Web.
  • invented by Tim Berners-Lee
  • jointly developed by the W3C and the IETF
  • version history :
    • HTTP 0.9 (1991)
    • HTTP 1.0 (RFC 1945, May 1996) ;
    • HTTP 1.1 (RFS 2068 Jan 1997, RFC 2116 Jun 1999), RFC 7230 - 7235 (Jun 2014) ;
    • HTTP 2 (RFC 7540 May 2015) ;

HTTP Fundamentals

  • HTTP runs on top of TCP/IP, using TCP port 80 by default or TCP port 443 for HTPPS (over SSL/TLS)
  • HTTP is based on a client - server architecture
    • Client, a.k.a. user agents (UA) : request resources from server.
    • Web Client : ex. web browsers such as Chrome, Edge, Firefox .. Web crawlers/spiders : ex. googlebot
    • http Client : ex. applications in mobile
      • web browser, web crawlers/spider, other end user tools and applications
    • Servers :
      • Origin Server - web resources are physically there. - a machine w/ application running through web.
      • Proxy Servers (to authenticate), gateways, tunnels (blind relay)
  • HTTP uses a request-response standard protocol
    • the clients sends an HTTP request message to the server
    • the server processes the request and replies with an HTTP response message
  • HTTP is a stateless communications protocol
    • Servers do not keep information about clients in-between request
  • HTTP provides support for other functionalities, such as :
    • cache control
    • control media type (MIME) specification
    • language and character set specification
    • content/transfer coding
    • content negotiation
    • client-server protocol negotiations
    • persistent connection
    • request pipelining
    • authentication/authorization
    • etc.
