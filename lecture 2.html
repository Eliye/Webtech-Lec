Cascading Style Sheets (CSS)

CSS History

Cascading Style Sheet (CSS) is a simple mechanism for adding style (e.g., fonts, colors, spacing) to Web documents.

Time Table
  • October 1994: Hakon Wium Lie released the first draft of "Cascading HTML Style Sheets"
  • May 1995: The discussions on the www-style mailing list
  • August 1996: Microsoft Internet Explorer became the first browser to support CSS.
  • December 1996: The CSS Level 1 W3C Recommendation was made
  • The release of CSS 1 supported:
    • font properties
    • text attributes
    • alignment of text
    • tables
    • images and more
    • colors of text and backgrounds
    • spacing of words
    • letters and lines
    • margins
    • borders
    • padding and positioning
    • and unique identification and classification of groups of attributes
  • May 1998: the W3C released CSS 2 which added new capabilities including z-index, media types, bidirectional text, absolute, relative and fixed positioning, and support for aural style sheets.
  • Between June 2011 and June 2012: the following four modules were released as formal recommendations: color, selectors level 3, namespaces and media queries.
Versions

CSS Rule Sets

Consist of a selector followed by a brace enclosed declaration block, which contains zero.

    /* Comment */
    p {
        color: green;
    }
    // Invalid Comment

p - selector

{} - declaration block

: - property separator

color - property name

green - property value

/* or */ - comment

// - invalid comment

CSS Selectors
Selector

Structure used as a condition in a CSS rule to determine which elements in the document tree are matched by the selector and are thus targeted by the formatting specified in the CSS rule declaration block. The matched elements are called the subjects of the selector.

Selector Syntax

Chain of one or more sequences of simple selectors separated by combinators (>, +) with one pseudo-element possibly appended to the last sequence.

div#abc>p.xyz[title]­+span:last-of-type::­after
    
    div             - type selector
    #abc            - ID Selector
    >               - Combinators
    p               - paragraph (type)
    .xyz            - class selector
    +               - adjacent
    span            - type selector
    :last-of-type   - pseudo class
    ::after         - pseudo element
Sequence of Simple Selectors

Chain of simple selectors not separated by combinators.

Always starts with a type selector or a universal selector.

Cannot contain other type selectors or universal selectors.

Group of Selectors

Comma-separated list of selectors representing the union of all elements selected by each of the selectors in the list.

Simple Selectors
  • Type selector - type of targeting elements
    •     p {         
        font-size: 16px;
    }
  • Universal selector - represented by an asterisk (*)
    •     * {
        padding-top: 50px;
    }
  • Attribute selector
    • [attr]
    • [attr = value]
    • [attr == value]
    • [attrl = value]
    • [attr^ = value]
    • [attr$ = value]
    • [attr* = value]
  • Class Selector - represented by a period
    •     .x {
        padding-right: 20px;
    }
  • ID Selector - pound sign
    •     #a {
        padding-left: 20px;
    }
Pseudo-class

Implicit class that is associated with the inherited classes or mark up.

  • Dynamic pseudo-class
    • Link psuedo-classes
      • :link
      • :visited
    • User action pseudo-class
      • :target
  • User action pseudo-classes
    • :hover - roll over effect, mouse pointer hovers
    • :active - activate a particular element by clicking a button
    • :focus - form many input element, focus is moved
  • Target pseudo-class
    • :target - target element
  • Language pseudo-class
    • :lang()
  • UI element states pseudo-classes
    • :enabled
    • :disabled
    • :checked
    • :indetermined
  • Structural pseudo-classes
    • :root
    • :first-child
    • :last-child
    • :only-child
    • :nth-child
    • :nth-child(an + b)
    • :nth-last-child()
    • :first-of-type
    • :last-of-type
    • :only-of-type
    • :nth-of-type
    • :nth-last-of-type - count from the last child
    • :empty
  • Negation pseudo-class
    • :not() - negates the logic of a selector
Combinators

Descendant combinator (whitespace, i.e space, tab, linefeed, carriage return, form feed)

  • child combinator (>) direct/immediate descendant
  • sibling combinator (same parent)
    • adjacent sibling combinator (+)
    • general sibling combinator (~)
Pseudo-elements
  • ::first-letter - :first-letter
  • ::first-line - :first-line
  • ::before - :before
  • ::after - :after
  • * counter-increment
  • * counter-reset
CSS Rules of Precedence
By origin and importance
  • user agent important declarations
  • user important declarations
  • author important declarations
  • author normal declarations
  • user normal declarations
  • user agent normal declarations
By specificity
  • inline style
  • number of id selectors
  • number of class selectors, attribute selectors and pseudo classes
  • number of type selectors and pseudo elements
By order
