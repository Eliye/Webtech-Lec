LESSON 3: Hypertext Resource Addressing (HTTP)

HTTP

  • HTTP resources are identified using URIs(RFC 3986) or more specifically, HTTP URLs.
    • Scheme (http or https)
    • Authority
      • user information or authentication credentials
      • host
        • domain name (resolved to an IP address using DNS(Domain Name System) of the server where the resource resides (or will be created))
      • port number
    • Path to resource (resolved relative to the document root on the server)
      • may refer to a static or dynamic resource
    • Query
      • typically provided as key = value pairs, with ampersand (&) separators between key/value pairs
      • may be URL -encoded
    • fragment identifier
