HTTP Request Methods

Standard Methods
GET

This is the most commonly used method. It is used to retrieve information from the given server using a given URI. Requests using GET should only retrieve data and should have no other effect on the data.

HEAD

Same as GET, except that the entity is not included in the reponse (i.e returns only the status line and header returned by a request without the message body. This is used to retrieve metadata about the entity implied by request without transferring the entity itself (e.g to test for link validity or resource modification). Like GET, it is supported by all general-purpose servers.

POST

Perform resource-specific processing of the entities enclosed in the message body by the resource.

PUT

Store the enclosed entity in the message body under the specified Request URI.

DELETE

Remove the resource associated with the request URI.

OPTIONS

Request information about the communication options available for the resource identified by the Request URI.

TRACE

Request a loop-back of the request message (ie., request the server to echo back to the client the received request message. Typically used for testing or diagnostics of the request or response chain.

CONNECT

Request the establishment of a tunnel to the destination origin server, and if successful, thereafter restrict its behavior to blind-forwarding of packets in both direction until the tunnel is closed. Commonly used to create an end to end virtual connection through one or more proxies.

Extension Methods
Example: WebDAV (RFC 4918)

PROPFIND, PROPPATCH, MKCOL, COPY, MOVE, LOCK, UNLOCK

Safe Methods

GET and HEAD methods should not have the significance of taking an action other than retrieval.

Idempotent Methods

Identical requests is the same as for a single request.

Cacheable Methods

It is an HTTP response that can be cached, that is stored to be retrieved and used later, saving a new request to the server. Not all HTTP responses can be cached.
